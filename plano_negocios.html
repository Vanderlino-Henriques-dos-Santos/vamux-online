1. Sumário Executivo

O VAMUX é uma inovadora plataforma de mobilidade urbana baseada em aplicativo, desenvolvida para conectar passageiros e motoristas parceiros de forma eficiente, segura e justa. Nosso objetivo é revolucionar o transporte em [Nome da Cidade/Região Alvo], oferecendo uma alternativa superior aos modelos existentes que, muitas vezes, falham em equilibrar a rentabilidade dos motoristas com a qualidade e segurança do serviço para os passageiros.

Identificamos no mercado atual a crescente demanda por conveniência e segurança no transporte, aliada à insatisfação de motoristas com as altas taxas de comissão e o suporte ineficaz das grandes plataformas. O VAMUX surge como a solução para essas lacunas, propondo um modelo de negócio mais equitativo para os motoristas e uma experiência de usuário superior, focada em segurança, transparência e personalização (se houver nicho específico).

Com uma arquitetura tecnológica robusta e escalável (Firebase e Google Maps Platform), o VAMUX está pronto para operar, oferecendo funcionalidades essenciais como cadastro e login intuitivos, solicitação e acompanhamento de corridas em tempo real para passageiros, e gestão de disponibilidade e aceitação de corridas para motoristas. 4. Produtos e Serviços (O Aplicativo VAMUX)

O VAMUX é uma plataforma de mobilidade baseada em interfaces web modernas e responsivas, que se comportam como aplicativos. O sistema é construído sobre HTML, CSS e JavaScript, utilizando o Firebase para um backend ágil e escalável, e o Google Maps Platform para todas as operações de geolocalização e rotas. As principais telas do aplicativo são projetadas para proporcionar uma experiência intuitiva e eficiente para passageiros e motoristas.

4.1. Página Inicial: Login e Cadastro (`index.html`)

Função Principal no Projeto: Esta é a porta de entrada universal para o VAMUX. Ela serve como o primeiro ponto de contato para todos os usuários, gerenciando o fluxo de autenticação e registro, e direcionando-os para suas respectivas interfaces após o login bem-sucedido.

Descrição Detalhada: A tela `index.html` apresenta uma interface limpa e intuitiva, com formulários que se alternam entre as opções de Login e Cadastro.

  • Funcionalidades Chave da Tela de Cadastro:
    • Nome Completo: Campo para identificação do usuário.
    • E-mail e Senha: Credenciais para acesso seguro, gerenciadas pelo Firebase Authentication. A senha requer um mínimo de 6 caracteres para segurança.
    • Seleção de Perfil: Botões de rádio ou seleção (Cadastrar como Passageiro / Cadastrar como Motorista). Esta escolha é fundamental, pois define o tipo de conta no Firebase (campo type na coleção users) e o redirecionamento pós-login para passageiro.html ou motorista.html.
    • Validação em Tempo Real: Feedback imediato sobre o preenchimento correto dos campos (ex: formato de e-mail inválido, senha muito curta).
    • Link para Login: Facilita a navegação para usuários que já possuem uma conta.
  • Funcionalidades Chave da Tela de Login:
    • E-mail e Senha: Campos para as credenciais já cadastradas.
    • Autenticação Firebase: Utiliza o Firebase Authentication para verificar as credenciais e garantir o acesso seguro ao sistema.
    • Redirecionamento Condicional: Após o login bem-sucedido, a aplicação verifica o tipo de usuário no Firebase Realtime Database (/users/{uid}/type) e redireciona automaticamente para passageiro.html ou motorista.html, oferecendo uma experiência personalizada.
    • Tratamento de Erros: Exibe mensagens claras para credenciais inválidas ou outros problemas de login.
    • Link para Cadastro: Permite que novos usuários criem uma conta caso tentem logar sem ter uma.

4.2. Interface do Passageiro (`passageiro.html`)

Função Principal no Projeto: Esta é a interface central para o usuário que busca transporte. Sua principal função é permitir que o passageiro solicite uma corrida de forma intuitiva, visualize sua localização e acompanhe o status da viagem em tempo real.

Descrição Detalhada: Ao acessar passageiro.html (após login como passageiro), o usuário é apresentado a uma interface dominada por um mapa interativo.

  • Funcionalidades Chave do Mapa e Localização:
    • Exibição de Mapa: Carrega o Google Maps (via Maps JavaScript API) centralizado na localização atual do usuário (se a permissão de geolocalização for concedida).
    • Marcadores: Exibe um marcador na localização de partida do passageiro e, após a solicitação, outro para o destino.
    • Rastreamento de Motorista: Uma vez que um motorista aceita a corrida, o mapa atualiza a posição do motorista em tempo real, permitindo que o passageiro o acompanhe visualmente.
  • Funcionalidades Chave de Solicitação de Corrida:
    • Campos de Origem e Destino: Input fields para o passageiro digitar o endereço de partida e o destino desejado.
    • Autocompletar Endereços (Google Places Autocomplete): Sugere endereços à medida que o usuário digita, melhorando a precisão e a velocidade de preenchimento.
    • Cálculo de Estimativa: Ao definir origem e destino, o sistema utiliza o Google Directions API para calcular e exibir a distância, o tempo estimado de viagem e o valor aproximado da corrida antes da solicitação. Isso oferece transparência e permite ao passageiro tomar uma decisão informada.
    • Botão "Solicitar Corrida": Envia o pedido de corrida para o Firebase Realtime Database, onde fica aguardando a aceitação de um motorista.
  • Funcionalidades Chave de Acompanhamento de Status:
    • Status da Corrida: A interface exibe mensagens dinâmicas sobre o status atual da corrida (ex: "Procurando motorista...", "Motorista a caminho!", "Corrida iniciada", "Corrida finalizada").
    • Informações do Motorista: Após a aceitação, pode exibir o nome do motorista, modelo do carro e placa.
    • Botão "Cancelar Corrida": Opção para o passageiro cancelar a corrida antes ou durante a viagem, com feedback para o motorista e atualização no Firebase.

4.3. Interface do Motorista (`motorista.html`)

Função Principal no Projeto: A interface do motorista é a ferramenta de trabalho para os parceiros VAMUX. Ela permite que o motorista gerencie sua disponibilidade, receba e gerencie solicitações de corrida e atualize sua localização em tempo real para o sistema.

Descrição Detalhada: Ao acessar motorista.html (após login como motorista), o profissional tem controle total sobre sua operação.

  • Funcionalidades Chave de Disponibilidade:
    • Botão "Ficar Online/Offline": Este toggle permite ao motorista indicar sua disponibilidade para receber corridas. Quando "Online", sua localização é continuamente enviada para o Firebase (/drivers_location/{uid}), tornando-o visível para os passageiros que solicitam corridas próximas. Quando "Offline", ele não receberá novas solicitações.
  • Funcionalidades Chave de Gestão de Solicitações:
    • Lista de Novas Corridas: Exibe em tempo real uma lista de solicitações de corrida pendentes (status pending) que estão dentro de sua área de atuação ou que o sistema sugere. Cada item da lista mostra detalhes essenciais: origem, destino, valor estimado da corrida e informações básicas do passageiro.
    • Botões "Aceitar" / "Recusar": Para cada solicitação recebida, o motorista pode aceitar (mudando o status da corrida para accepted no Firebase e vinculando-a ao seu UID) ou recusar.
  • Funcionalidades Chave de Corrida Ativa:
    • Visualização de Rota: Após aceitar uma corrida, o mapa exibe a rota otimizada para buscar o passageiro (do motorista até a origem do passageiro) e, posteriormente, a rota da origem ao destino final.
    • Informações da Corrida Ativa: Exibe detalhes completos da corrida atual: passageiro, origem, destino, valor.
    • Botão "Iniciar Corrida": Uma vez que o passageiro embarca, o motorista clica neste botão para mudar o status da corrida para started no Firebase, iniciando oficialmente a viagem e o cálculo final (se houver).
    • Botão "Finalizar Corrida": Ao chegar ao destino, o motorista clica para mudar o status da corrida para completed no Firebase, encerrando a viagem. Este acionamento pode, futuramente, disparar o cálculo do valor final e a transação de pagamento.
    • Botão "Cancelar Corrida": Permite ao motorista cancelar uma corrida ativa (com justificativa), atualizando o status para cancelled e notificando o passageiro.
  • Funcionalidade de Rastreamento GPS: O script motorista.js constantemente envia a geolocalização do motorista para o Firebase (/drivers_location), permitindo que os passageiros acompanhem o trajeto do motorista aceito.

4.4. Diferenciais do VAMUX

  • Comissão Justa para Motoristas: O VAMUX se posiciona como um parceiro real para os motoristas, oferecendo uma das menores taxas de comissão do mercado (ex: 15-20% do valor da corrida). Isso significa que uma maior parcela da receita permanece com o motorista, incentivando a adesão e a lealdade, e resultando em um serviço de maior qualidade.
  • Foco na Segurança Integrada: A segurança é um pilar central. Além das verificações de antecedentes rigorosas para motoristas e veículos, o aplicativo incorporará funcionalidades como:
    • Botão de Pânico: Disponível para passageiros e motoristas, conectando-los diretamente com autoridades ou contatos de emergência configurados.
    • Compartilhamento de Rota: Passageiros poderão compartilhar os detalhes de sua viagem em tempo real com amigos e familiares.
    • Sistema de Avaliação Bidirecional: Permite que motoristas e passageiros avaliem uns aos outros, criando um ambiente de responsabilidade mútua e incentivando boas práticas.
  • Suporte Humanizado e Local: Ao contrário do suporte automatizado e impessoal das grandes plataformas, o VAMUX oferecerá um atendimento ao cliente ágil, eficaz e humanizado. Contaremos com uma equipe de suporte local, disponível via chat no aplicativo, telefone e e-mail, garantindo que as dúvidas e problemas sejam resolvidos de forma rápida e atenciosa.
  • Potencial de Segmentação de Mercado (Nicho): A arquitetura flexível do VAMUX permitirá a fácil introdução de categorias de serviço especializadas, atendendo a demandas específicas do mercado de [Nome da Cidade/Região], tais como:
    • VAMUX Kids: Motoristas com treinamento específico para transporte de crianças, veículos inspecionados e assentos de segurança.
    • VAMUX Pets: Serviço de transporte adaptado para usuários com animais de estimação.
    • VAMUX Eco: Opções de veículos elétricos ou híbridos, alinhando-se à crescente preocupação com a sustentabilidade.
    • VAMUX Acessível: Veículos adaptados e motoristas treinados para atender pessoas com mobilidade reduzida, garantindo inclusão.
  • Programas de Fidelidade e Recompensa: Desenvolveremos um sistema de pontos e níveis para passageiros frequentes, que poderão trocar pontos por descontos em corridas futuras ou benefícios em parceiros locais. Para motoristas, programas de incentivo baseados em desempenho e avaliações.

4.5. Resumo das Interfaces Principais

Para uma visão geral concisa, o projeto VAMUX é construído sobre três interfaces principais que guiam passageiros e motoristas através de todo o ciclo de uma corrida:

  • Página Inicial: Login e Cadastro (`index.html`): A porta de entrada universal que gerencia o acesso à plataforma. Permite que novos usuários se cadastrem (como passageiro ou motorista) e que usuários existentes façam login, redirecionando-os para a interface apropriada. Essencial para segurança e personalização.
  • Interface do Passageiro (`passageiro.html`): A central de comando para solicitação de transporte. Permite ao passageiro definir origem/destino, obter estimativas de preço e tempo, solicitar a corrida e acompanhar o motorista em tempo real no mapa, garantindo conveniência, transparência e segurança.
  • Interface do Motorista (`motorista.html`): O painel de trabalho para os parceiros. Permite ao motorista gerenciar sua disponibilidade (online/offline), receber e aceitar novas solicitações de corrida, visualizar rotas e controlar o status da viagem (iniciar, finalizar, cancelar), otimizando sua produtividade e garantindo autonomia.

Juntas, estas interfaces formam um ecossistema de mobilidade completo e eficiente, garantindo comunicação fluida e um serviço de transporte sob demanda seguro e justo para todos os envolvidos.
